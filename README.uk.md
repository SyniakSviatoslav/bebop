# ◈ Bebop

> **Твоя кухня, твій корабель, твій зріз.**
> Автономний CLI-агент для написання коду, що керує **будь-яким** підʼєднаним агентичним CLI —
> Claude Code, Codex, OpenCode, Hermes, Aider, Goose — за єдиним ядром-охоронцем, із
> **безкоштовними LLM за замовчуванням** та простим перемикачем на будь-який із них.

![Живий сеанс Bebop CLI — справжній запис asciinema](docs/footage/bebop-session.gif)
> *Реальний запис: живий сеанс `bebop`, знятий через [asciinema](https://asciinema.org), перетворений
> на GIF утилітою [agg](https://github.com/asciinema/agg). Показує справжній вивід `boot`
> (самосертифікація охоронця), `status` та `use native` — без постановки, без підробок.
> Вихідний cast: [`docs/footage/bebop-session.cast`](docs/footage/bebop-session.cast).*

Bebop — повністю незалежний інструмент. Його власний кордон довіри (ядро-охоронець на Rust/WASM),
власний ретрівер (VSA), власний маршрутизатор токенів і власний копілот (doer→checker) живуть
**у цьому репозиторії** — жодного іншого проєкту не потрібно.

- **Ліцензія:** AGPL-3.0 · **Середовище:** Node 22 LTS (для CLI не потрібен `better-auth`)
- **Бренд:** Warm Cosmo-Noir. Основний сигнальний колір: корабельний бірюзовий `#46B0A4`.
- **Політика:** кожне твердження в репозиторії підкріплене живим тестом або детермінованим
  тестом. Див. [`docs/RULES.md`](docs/RULES.md) — **Постійний сумнів: жодного твердження без
  перевірки.**

> 📖 **Час читання:** ~9 хв · 🎧 **Слухати:** [озвучення README (mp3)](docs/narration/README-narration.mp3)
> + [транскрипт](docs/narration/README-narration.md) · 🤖 **Для ботів/агентів:** [`llms.txt`](llms.txt)
> та структурований [`llm-manifest.json`](llm-manifest.json).

**Ключові висновки (прочитайте це, якщо більше нічого):**
1. Bebop — **один штурвал для кожного кодувального агента** — перемикайте агентів однією командою;
   ваші охоронці, памʼять і хуки переміщуються разом із вами (без привʼязки).
2. Охоронець — **ваш і перевірений** — ядро на Rust/WASM забороняє auth/money/secrets/migrations,
   якщо людина не схвалить; `bebop boot` доводить, що він спрацьовує.
3. **Безкоштовно за замовчуванням** — стартує на безкоштовному рівні OpenRouter; при відсутності
   ключа працює keyless-цикл, тож вас ніколи не заблоковано.
4. **Кожне твердження доки довідне** — зробіть fork і запустіть `npm test` + зонди з
   [`docs/VERIFICATION-MATRIX.md`](docs/VERIFICATION-MATRIX.md).
5. **Чесність щодо обмежень** — не провайдер моделей, не пісочниця, не GUI, не мультиюзер. Див.
   [Чим Bebop НЕ є](#чим-bebop-не-є).

## Історія (розкажіть її пʼятирічній дитині)

> *Кожен great фільм про пограбування починається однаково: одна команда, багато фахівців, і план,
> що виживає лише тому, що хтось стереже двері.*

Уяви, у тебе є **робот-помічник**, що пише компʼютерний код. Але кожен робот-помічник має свої
власні правила, свій пульт і свій улюблений спосіб робити речі. Якщо перейти від одного робота до
іншого — доведеться вчити зовсім новий спосіб розмови.

**Bebop — це один штурвал, що підходить до кожного робота.** Ти сідаєш у крісло Bebop, і Bebop
знає, як керувати роботом-Claude, роботом-Codex, роботом-OpenCode та іншими. Кажeш «поїхали» —
і Bebop вирішує, якому роботу доручити роботу, зазвичай маленькому безкоштовному, якщо завдання
не складне.

І в Bebop є **охоронець**. Перш ніж будь-який робот торкнеться твоїх файлів, охоронець перевіряє:
*«Це шухляда з грішми? Це шухляда з секретами? Це шухляда „не чіпай“?»* Якщо так — роботу
**не дозволено** — хіба що дорослий скаже, що можна. Охоронець ніколи не втомлюється і нічого не
забуває. Тому Bebop можна довірити свою кухню.

І ще — **найкращий робот безкоштовний**. Bebop стартує на безкоштовній моделі, тож будувати можна
ще до того, як заплатиш хоч копійку.

## Чому бізнесу це справді важливо (біль, яку знімає Bebop)

> *Кожна компанія, що купує агента, зрештою одружується з ним. Весілля дешеве; розлучення — ось де
> йдуть гроші.*

Більшість команд впроваджують агентичний CLI, а потім **замикаються** (lock-in): їхні скрипти,
хуки та правила-охоронці працюють лише з одним вендором. Перехід пізніше означає переписування
всього. Гірше того, рейки безпеки («комплаєнс») сховані у чорній скриньці вендора, тож ти не можеш
довести, що агенту було дозволено чи заборонено.

Bebop знімає цей біль чотирма конкретними способами:

| Біль сьогодні | Що робить Bebop |
|---|---|
| **Привʼязка до вендора** — твоя автоматизація розмовляє лише діалектом одного CLI | Одна панель керування для **кожного** підʼєднаного агента. Перемикай через `bebop use <agent>`; твої хуки, памʼять та охоронці переміщуються разом. |
| **Охорону не можна аудитувати** — «довірся нам» із чорної скриньки | **Охоронець із відмовою за замовчуванням, що належить тобі**: red-line globs (auth / money / secrets / migrations) + allow-лист області, забезпечені ядром на Rust/WASM, яке можна читати й тестувати. `bebop boot` доводить, що ворота забороняють на червоному. |
| **Сюрприз із вартістю** — стрибки рахунків від завжди-ввімкнених преміум-моделей | **Безкоштовний LLM за замовчуванням**; маршрутизатор надсилає кожне завдання у *найдешевший адекватний* ланцюг. Немає ключа? Keyless-цикл усе одно працює, тож тебе не заблоковано. |
| **Неперевірювані твердження** — «працює, обіцяємо» | **Постійний сумнів**: кожен рядок доки підкріплений живим тестом або детермінованим ([`docs/VERIFICATION-MATRIX.md`](docs/VERIFICATION-MATRIX.md)). Зроби fork і перевір доказ сам. |

Підсумок: Bebop перетворює «ми сподіваємося, що агент безпечний і дешевий» на «ми можемо
**довести**, що охоронець спрацював, а дешева модель взяла нудні 80%».

## Швидкий старт

> *Досить слів. Ось корабель; ось ключі.*

```bash
git clone https://github.com/SyniakSviatoslav/bebop
cd bebop
npm install
npm run build        # компілює ядро-охоронець Rust/WASM → src/bebop_core.wasm
npm link             # або просто: npx tsx bebop.ts <cmd>
```

```bash
bebop boot                                          # самотест охоронця — відмовляє стартувати, якщо ворота не можуть піти в RED
bebop status                                        # ротація агентів + що підʼєднано
bebop agents                                        # кожен агентичний CLI, яким керує Bebop, наживо
bebop use native                                    # (за замовчуванням) keyless-цикл — працює без жодного ключа
bebop dispatch "fix the red ship animation"        # запускається за охоронцем + копілотом
bebop run doer                                      # повний цикл (детермінований native-заглушка за замовчуванням)
```
> `native` — за замовчуванням і не потребує ключа. Щоб увімкнути рівень **free**, встанови
> `OPENROUTER_API_KEY` (або `OPENROUTER_FREE_KEY`) і виконай `bebop use free` (без ключа він
> відмовляє — це відмова-за-замиканням, за задумом).

> WASM-ядро (`src/bebop_core.wasm`) закомічене, тож CLI працює без Rust-тулчейну.
> Перебудуй: `cd crates/core && bash build.sh`.

## Команди

> *Панель керування. Кожен важіль підписано — без таємничих перемикачів.*

| Команда | Що робить |
|---|---|
| `boot` | Самотест охоронця. Red-line + scope-ворота мусять забороняти на поганому і пропускати на доброму. Інакше відмовляє стартувати. |
| `status` | Ротація агентів + статус зʼєднання (free першим за замовчуванням). |
| `agents` | Перелік **кожного** агентичного CLI, яким керує Bebop, зі статусом і командою перемикання. |
| `use <backend>` | Перемкнути агента за замовчуванням і зберегти. Відмовляє непідʼєднаному бекенду, хіба що `--force`. |
| `run [doer\|reason\|redline]` | Повний агентичний цикл. Маршрутизує клас завдання у найдешевший адекватний ланцюг моделі. |
| `dispatch "<task>"` | Разове завдання через охоронця + копілота. Red-line завдання **заборонені до запуску будь-якого агента**. |
| `route <class>` | Показати рішення маршрутизатора токенів для класу завдання. |
| `recall <query>` | Запит лише до in-process живої памʼяті. VSA-ретрівер **ще не включено** — `recall` чесно про це каже, а не підробляє результат. |
| `govern "<0.9,0.6,...>"` | L5 телеметрійний губернатор (PID-авторитет + ICIR + резонанс) поверх потоку якості. |
| `self [maintain\|evolve\|session\|loop]` | Самообслуговування / самоеволюція (відмова-за-замиканням, оборотне). |
| `node` | Ідентичність вузла, шифрована at rest (PQ + Ed25519). |
| `mcp` | Model Context Protocol сервер поверх stdio (нуль нових залежностей). |
| `init` | Майстер персоналізації на 5 осей → `~/.bebop/settings.json`. |
| `map [module]` | «Зрозумій усе»: рендер реального графа імпортів як SVG. |
| `diagrams` | Перегенерувати кожну візуалізацію (мапа проєкту + концептуальні схеми фіч). |
| `help` | Цей список. |

## Як влаштовано (коротко)

> *Під капотом Bebop — це два шари, складені разом: оболонка, що говорить, і ядро, що памʼятає, —
> з охоронцем між ними.*

Bebop — це оболонка на TypeScript поверх **ядра-охоронця Rust/WASM**. Оболонка володіє
загальною політикою — охоронець (red lines + область), маршрутизатор токенів, копілот
(doer→checker), памʼять (VSA + живі знання) і L5-губернатор. Агенти — тонкі адаптери, через які
проходить кондукт. Ядро (`crates/core`, скомпільоване у `bebop_core.wasm`) — це C-ABI модуль,
написаний вручну, **без wasm-bindgen**; TS-завантажувач інстанціює його з нульовими залежностями,
а охоронець делегує йому, інакше відкочується до вірної TS-копії.

Довга версія з діаграмами — у вікі: [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) ·
[`docs/features/`](docs/features/) · [`docs/VERIFICATION-MATRIX.md`](docs/VERIFICATION-MATRIX.md).

## Bebop проти інших (чесне порівняння)

> *Жоден інструмент не герой кожної історії. Ось де веде Bebop — і де варто обрати когось іншого.*

Догма правди бʼє в обидва боки: Bebop — **не** магія, і він **не** завжди right tool. Ось як він
порівнюється з агентичними CLI та обгортками, що їх реально використовують.

| | Bebop | Claude Code | Codex CLI | OpenCode | Aider | Goose | звичайний `aishell`/обгортка |
|---|---|---|---|---|---|---|---|
| **Керує кількома агентами** | ✅ будь-який підʼєднаний CLI | ❌ лише Anthropic | ❌ лише OpenAI | ❌ своя модель | ❌ model-specific | ❌ своя | ⚠️ скриптовано, без охоронця |
| **Безкоштовний LLM за замовчуванням** | ✅ безкоштовний рівень OpenRouter | ❌ платно | ❌ платно | ⚠️ залежить | ⚠️ залежить | ⚠️ залежить | ⚠️ залежить |
| **Охоронець, що належить тобі й тестується** | ✅ Rust/WASM, `boot` доводить | ⚠️ налаштування вендора | ⚠️ політика вендора | ⚠️ конфіг | ⚠️ не вбудовано | ⚠️ конфіг | ❌ немає |
| **Відмова-за-замиканням на red lines** | ✅ забороняє auth/money/secrets | ⚠️ дозволи | ⚠️ політика | ⚠️ конфіг | ❌ | ⚠️ конфіг | ❌ |
| **Доки, перевірені математикою** | ✅ кожен рядок протестовано | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **Візуальна мапа проєкту** | ✅ `bebop map` / `diagrams` | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **Автономний репозиторій (без dowiz)** | ✅ | n/a | n/a | ✅ | ✅ | ✅ | ✅ |
| **Вартість перемикання агентів** | низька (одна команда) | висока | висока | висока | висока | висока | низька, але небезпечна |

**Де веде Bebop:** тобі потрібна одна аудитована площина керування кількома агентами, ти мусиш
*довести*, що охоронець спрацював, хочеш безкоштовне за замовчуванням і доки, які можеш
переперевірити fork-ом.

**Де веде інший інструмент:** якщо ти завжди використовуєш одну модель вендора і ніколи не
перемикаєшся — рідна CLI вендора може здатися щільнішою (Bebop додає шар непрямості). Якщо потрібен
багатий хостed UI або першокласна enterprise-підтримка — продукт вендора бʼє CLI-обгортку.

## Чим Bebop НЕ є

> *Кожен чесний продукт постачається зі списком «не роби». Ось наш — без прикрас.*

- **Bebop — НЕ провайдер моделей.** Він не тренує, не хостить і не обслуговує LLM. Він *керує*
  моделями через агентів/адаптерів. Якість виводу повністю залежить від агента/бекенду, на який ти
  вказуєш.
- **Bebop — НЕ пісочниця.** Ядро-охоронець забороняє *відомі* red-line шляхи (globs) та поза-областю,
  але це політична брама, не OS-контейнер. Воно не зупинить підʼєднаного агента від чогось злого
  *поза* відомими шляхами. Запускай агентів із найменшими привілеями.
- **Bebop — НЕ ретрівер живих знань (поки що).** `recall` питає лише in-process живу памʼять.
  VSA-ретрівер описано в доках, але **не включено** у репозиторій; `recall` чесно про це каже, а не
  підробляє результати.
- **Bebop — НЕ GUI.** Це термінальний CLI. Немає десктоп-застосунку чи веб-дашборду (сервер `sync` —
  це dev-транспорт, не продуктовий UI).
- **Bebop — НЕ заміна людській перевірці змін money/auth/RLS.** Охоронець *відмовляє* їм за
  замовчуванням; коли людина схвалює — людина відповідає за наслідок. «Заборонено, якщо не схвалено»
  — це швидкісний барʼєр, не гарантія.
- **Bebop — НЕ мультиюзер / не хостований сервіс.** Один користувач, один `~/.bebop/settings.json`,
  локальний процес. Жодних акаунтів, спільної оренди чи хмарної площини керування.
- **Bebop — НЕ система збірки, пакетний менеджер чи CI-ранер.** Він оркеструє агентів; він не
  замінює `npm`/`cargo`/`git`/GitHub Actions (через агентів він їх викликає).
- **Bebop НЕ гарантує успіху завдання.** Копілот (doer→checker) і маршрутизатор зменшують ризик; вони
  не роблять агента всезнаючим. Погані промпти досі дають поганий код.

## Перевірка (що справді працює)

> *У Bebop твердження без чека — лише чутка. Ось чеки.*

Кожне твердження вище перевіряється тест-сьютом і живим зондуванням — див.
[`docs/VERIFICATION-MATRIX.md`](docs/VERIFICATION-MATRIX.md) для повної таблиці доказів із 35 рядків.

- **Охоронець RED+GREEN:** `bebop boot` сертифікує, що ворота забороняють на червоному, пропускають
  на зеленому. Тест ядра забороняє `auth/token` і дозволяє `tools/bebop/x.ts`.
- **Відмова dispatch справжня:** `bebop dispatch "edit packages/db/migrations/002_users.sql"` виходить
  ненульовим кодом із `⛔ DENIED by guard (rust)` (регресійний тест запускає справжній CLI і стверджує
  це).
- **Безкоштовне за замовчуванням справжнє:** `bebop status` показує `free → … → native`; з ключем
  `dispatch` робить справжній виклик OpenRouter.
- **Перемикання справжнє:** `bebop use native` зберігає його; `bebop use claude` (непідʼєднаний)
  відмовляє, хіба що `--force`.
- **Лічильники (перевірено, не вгадано):** `npm test` → **165** TS-тестів; `cargo test -p bebop-core`
  → **7** тестів ядра Rust; `npx tsc --noEmit` → 0 помилок.

## Переклад / Translations

Цей файл існує також українською: [**README.uk.md**](README.uk.md). GitHub покаже перемикач мов, коли
в корені є кілька README.<мова>.md. Будь-яку сторінку доки також можна перекласти рідним інструментом
перекладу браузера (Google Translate / Firefox Translations) — Bebop пише доки чистою семантичною
розміткою саме для цього.

Автопереклад іншими мовами (чернетки, не перевірені!) генерує скрипт
[`scripts/i18n-translate.mjs`](scripts/i18n-translate.mjs) через вільний open-source рушій
**LibreTranslate** (AGPL-3.0) або **Argos Translate** (Apache-2.0, повністю офлайн) — без ключів і
без телеметрії. Кодові блоки та посилання зберігаються дослівно.
