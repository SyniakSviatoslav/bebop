# Internationalization (i18n) for Bebop docs

## Does GitHub translate docs natively?
**No.** GitHub has no built-in auto-translation. Its only native lever is the
`README.<lang>.md` convention: when several `README.<lang>.md` files exist in the repo root,
GitHub shows a language switcher at the top of the rendered README. Everything else is external.

Bebop therefore does two things, both free and open-source:

1. **Browser-native translatability** — zero translation, always truthful. Docs are written in clean
   semantic markdown so any browser's built-in translator (Chrome/Edge → Google Translate, Firefox →
   Firefox Translations, which is fully local/OSS) can translate the rendered page on the fly.
2. **OSS auto-translate** — a script that generates *machine-draft* localized files via a real
   open-source engine, clearly stamped as unreviewed.

## 1. Localized README (native GitHub switcher)
- `README.md` — English (source of truth).
- `README.uk.md` — Українська (hand-reviewed).
- Add more: `README.<lang>.md` where `<lang>` is an ISO 639-1 code (`zh`, `es`, `de`, …). GitHub picks
  them up automatically.

## 2. OSS auto-translate script
`scripts/i18n-translate.mjs` (Node, zero npm deps — uses built-in `fetch`) talks to a self-hostable
open-source engine:

| Engine | License | How |
|---|---|---|
| **LibreTranslate** | AGPL-3.0 | Default. Point `LT_URL` at your instance (or `http://localhost:5000`). |
| **Argos Translate** | Apache-2.0 | Fully offline. If `argos-translate` CLI is on PATH and `LT_URL` is unset, it's used. |

Self-host once (free):
```bash
docker run -d -p 5000:5000 libretranslate/libretranslate
# or, offline: pip install argos-translate && argos-translate --install en_uk
```

Usage:
```bash
node scripts/i18n-translate.mjs --lang uk                 # README.md -> README.uk.md
node scripts/i18n-translate.mjs --file docs/ARCHITECTURE.md --lang uk
node scripts/i18n-translate.mjs --all uk zh es            # many languages
```

The script:
- preserves fenced ```` ``` ```` code blocks, inline `` `code` ``, and `[links](url)` verbatim (never
  sent for translation);
- stamps every output with a `MACHINE TRANSLATED DRAFT` banner;
- sends **no keys, no telemetry** to any proprietary service.

> **Constant Doubt:** machine output is NOT authoritative. Review technical claims before trusting a
> generated file. The English source remains the single source of truth.

## Why not a hosted SaaS translator?
Hosted "free" translation APIs now require API keys and phone verification, and they are not
open-source. Bebop uses only self-hostable OSS engines so the docs pipeline stays free, auditable, and
offline-capable — consistent with the project's "own your tooling" ethos.
