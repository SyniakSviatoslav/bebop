# Bebop — limitations narration transcript

> Spoken version of the "What Bebop is NOT" section. Recorded from `docs/narration/limitations-narration.mp3`.

Bebop is not magic, and it is honest about that.

It is not a model provider — it drives models, it does not train or host them, so the quality you get
depends on the agent you connect.

It is not a sandbox — the guard is a policy gate that blocks known dangerous paths, not an
operating-system cage; run agents with the least privilege they need.

It is not a living-knowledge retriever yet; the recall command is honest that the embedding retriever
is not bundled.

It is not a graphical app — it is a terminal tool with no desktop or web dashboard.

It is not a replacement for a human reviewing money, auth, or database-migration changes; when a
human approves, the human owns the consequence.

It is not multi-user and not a hosted service — one person, one local process, no accounts.

And it does not guarantee success — the copilot lowers risk but does not make a robot all-knowing.
Bad instructions still make bad code.
